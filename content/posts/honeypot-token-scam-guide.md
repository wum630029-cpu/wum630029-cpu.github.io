## 一、什么是蜜罐代币和貔貅币？

### 蜜罐代币（Honeypot Token）

**蜜罐代币是一种恶意智能合约，允许用户买入代币，但通过各种手段阻止用户卖出或转账。** 这个名字来自「蜜罐」（Honeypot）——它用诱人的涨幅吸引你「掉进去」，然后让你无法脱身。

蜜罐的本质是**权限不对等**——合约创建者（部署者）可以随时卖出或管理代币，但普通用户不行。

### 貔貅币（Token with Sale Restriction）

貔貅币是蜜罐代币在中文社区中的俗称——传说貔貅没有肛门，只吃不排泄，和这类代币「只进不出」的特性完美呼应。

### 它们和普通 Rug Pull 的区别

| 类型 | 运作方式 | 时间线 | 用户损失 |
|:----|:--------|:-----|:--------|
| **貔貅币（Honeypot）** | 合约中设置了卖出限制，用户无法卖出 | 可持续数天到数周 | 买入资金全部被锁 |
| **Rug Pull** | 创建者撤走流动性，代币价格归零 | 几分钟到几小时 | 代币变废纸 |
| **貔貅 + Rug Pull** | 先限制用户卖出，等用户买够后直接撤池 | 混合模式 | 双重打击 |

> ⚠️ 现实情况往往是「貔貅币 + Rug Pull」的组合拳：前期让你买得进卖不出，等你大量买入后，创建者直接抽走流动性池中的所有资金，代币归零。

## 二、貔貅币的常见技术实现方式

了解貔貅币的技术原理，是识别它的第一步。

### 2.1 白名单限制（Whitelist / OnlyOwner）

这是最基础的貔貅实现方式。合约中把创建者的地址加入「白名单」，只有白名单地址才能卖出或转账。

```solidity
// 伪代码示例
function _transfer(from, to, amount) internal {
    // 如果 transfer 是从用户到 DEX（卖出），检查白名单
    if (to == pair) {
        require(whitelist[from] == true, "禁止卖出");  // 仅有创建者可卖
    }
    super._transfer(from, to, amount);
}
```

**检测难度**：⭐⭐ 容易被自动化工具发现

### 2.2 高买税 / 黑名单（High Tax / Blacklist）

合约设置极高的买卖税（如卖出税 99%），或者设置了一个黑名单——在你买入后将你的地址加入黑名单，禁止任何转账操作。

```solidity
// 伪代码：黑名单模式
function _transfer(from, to, amount) internal {
    require(blacklist[from] == false, "地址被限制");
    require(blacklist[to] == false, "地址被限制");
    super._transfer(from, to, amount);
}

function addToBlacklist(address[] memory addresses) external onlyOwner {
    for (uint i = 0; i < addresses.length; i++) {
        blacklist[addresses[i]] = true;   // 创建者可随时拉黑任意地址
    }
}
```

**检测难度**：⭐⭐⭐ 需要分析具体买卖税率

### 2.3 根据余额动态调整（Balance-Dependent）

这是最隐蔽的貔貅方式。合约在卖出时检查调用者的代币余额——如果余额超过一定数量则拒绝卖出。

```solidity
// 伪代码：余额限制
function _transfer(from, to, amount) internal {
    if (to == pair) {  // 卖出操作
        require(balanceOf[from] <= maxSellAmount, "单笔卖出超限");
    }
}
```

还有一种更狡猾的——**卖出数量取决于你的持仓时间**：买入不到 24 小时的用户不能卖出，或者卖出数量受限制。

**检测难度**：⭐⭐⭐⭐ 模拟交易不一定能发现

### 2.4 动态税率（Dynamic Tax）

合约中的卖出税率不是固定的，而是**由创建者通过后台控制的**。当创建者看到大量用户买入后，可以一键将卖出税从 5% 提高到 100%，所有人都无法卖出。

```solidity
// 伪代码：可调税率
uint256 public sellTax = 5;  // 初始 5%
function setSellTax(uint256 newTax) external onlyOwner {
    sellTax = newTax;  // 创建者随时可以改成 99 甚至 100
}
```

**检测难度**：⭐⭐⭐ 初始看起来正常，后期突然变脸

### 2.5 时间锁 / 函数开关（TimeLock / Pausable）

合约中包含交易暂停功能，创建者可以在任意时间点暂停所有转账操作。通常配合「跑路」使用——等用户买入足够多后，一键暂停合约，所有人资产被锁。

**检测难度**：⭐⭐⭐ 如果有防貔貅检测工具可以发现

## 三、如何识别貔貅币：实战检测方法

### 3.1 最可靠：使用专业检测工具

#### 🛠 Token Sniffer（最推荐）

[Token Sniffer](https://tokensniffer.com) 是最权威的代币安全检测平台。使用方法：

1. 复制代币合约地址
2. 粘贴到 Token Sniffer 搜索框
3. 查看检测报告

**重点关注指标：**

| 指标 | 安全范围 | 危险信号 |
|:----|:-------|:--------|
| **Honeypot** | ❌ No | ✅ Yes → 立即放弃 |
| **Mintable** | ❌ No/Revoked | ✅ Yes → 可无限增发 |
| **Trading Cooldown** | ❌ No | ✅ Yes → 可能有交易冷却限制 |
| **Owner Renounced** | ✅ Yes | ❌ No → 合约权限未放弃 |
| **Proxy Contract** | ❌ No | ✅ Yes → 合约可升级（可被修改逻辑） |
| **Hidden Owner** | ❌ No | ✅ Yes → 有隐藏的管理员地址 |
| **Anti-Whale** | 合理的限制 | 极端限制（每笔 < 0.1%） |

**评分参考（Token Sniffer 分数）：**
- **80-100/100**：较安全的代币（但不绝对）
- **50-79/100**：有一定风险，需要深入检查
- **0-49/100**：**高度危险，不建议交易**
- ⚠️ 评分高的不一定安全——有些貔貅合约可以绕过 Token Sniffer 的检测

#### 🛠 Honeypot.is（专用检测）

[Honeypot.is](https://honeypot.is) 是一个专门检测蜜罐代币的工具。它会模拟一笔「买入」和一笔「卖出」交易，看卖出是否成功。

1. 输入代币合约地址
2. 选择网络（Ethereum / BSC / Polygon 等）
3. 点击 Check
4. 查看结果

**结果解读：**

```
✅ No honeypot detected    → 通过基础检测（但仍不能 100% 保证）
❌ Honeypot detected       → 确定是貔貅币，绝对不要买
⚠️ Could not determine    → 检测失败，需要进一步分析
```

> 💡 **最佳实践**：同时用 **Token Sniffer** 和 **Honeypot.is** 检查——两个工具都通过，安全概率更高。

#### 🛠 GoPlus Security API

[GoPlus](https://gopluslabs.io/) 提供了代币安全检测数据库，可以通过其网站或接入其 API 进行检查。许多钱包（如 MetaMask、Rabby）已经集成了 GoPlus 的安全检测。

**钱包联动**：Rabby Wallet 等新式钱包默认会在交易前对代币合约做安全检测，如果有貔貅风险会直接弹窗警告。

#### 🛠 RugDoc

[RugDoc](https://rugdoc.io) 是一个社区安全评分平台，提供：
- 代币安全审计总结
- Rug Pull 预警
- 详细的安全分析方法

### 3.2 手工检测方法

如果不想依赖第三方工具，你也可以通过以下方式手动排查：

#### 方法 1：检查持有者分布

使用 **Etherscan** 或 **BscScan** 查看代币的持有者分布：

1. 在区块浏览器上搜索代币合约地址
2. 点击 **Holders** 标签
3. 观察前 10 名持有者的占比

```
危险信号：
- 排名前 3 的地址持有总量超过 50%
- 创建者地址持有大量代币
- 流动性池（Pair）地址外的地址持有量异常集中

安全信号：
- 持有者分布较均匀
- 流动性池地址是最大的持有者
- 创建者地址已放弃所有权（Ownership Renounced）
```

#### 方法 2：检查合约源代码

在区块浏览器上查看代币合约的源代码（如果已验证）：

1. 搜索代币合约地址
2. 点击 **Contract** 标签
3. 查看 **Read Contract** 中的函数

```
需要检查的关键函数：
- owner() / admin()      → 谁拥有合约？地址是否被放弃？
- renouncedOwnership()   → 是否已放弃所有权？
- maxTxAmount()          → 单笔最大交易量限制
- maxWalletAmount()      → 单个钱包最大持有量
- sellTax() / buyTax()   → 买卖税率
- blacklist()            → 是否有黑名单功能
- pause()                → 是否可以暂停交易

⛔ 如果合约未验证（No Contract Source）→ 高度危险！
```

#### 方法 3：Liquidity 检查

在 DexScreener 或区块浏览器上检查流动性：

```
需要关注的指标：
- 流动性数量：流动性池中的总锁仓价值
- 流动性锁定情况：LP Token 是否被锁定（Locked）在锁定合约中
- 流动性占比：代币总市值 vs 流动性池深度

危险信号：
- 流动性池中只有几百美元，但代币市值为几百万 → 随时可以 rug
- LP Token 未锁定，在创建者地址中 → 可以随时撤池
- 流动性池配对的是极低流动性代币（如 babydoge 之类）→ 难以真实估价
```

### 3.3 实战检查流程

**每次交易新代币之前，按以下流程检查：**

```
Step 1: Token Sniffer 检查合约
   ↓ 通过（≥ 60 分且 Honeypot = No）
Step 2: Honeypot.is 模拟卖出
   ↓ 通过（No honeypot detected）
Step 3: 检查持有者分布（Holder 集中度）
   ↓ 通过（分布均匀）
Step 4: 检查流动性（深度充足且 LP 锁定）
   ↓ 通过（流动性合理）
Step 5: 小额试仓（使用总预算 ≤ 5%）
   ↓ 确认能卖出
Step 6: 才考虑正式建仓

❌ 以上任何一步失败 → 立即放弃该代币！
```

> ⚠️ **重要提醒**：没有工具是 100% 可靠的。总有一些精心设计的貔貅合约可以绕过所有检测工具。这就是为什么**「先小额试仓，确认能卖出」**是最后一道防线。

## 四、交易中的实时预警信号

你正在 DexScreener 或类似平台上盯盘，下面这些信号应该立刻引起你的警觉：

### 🚨 高危信号（马上撤出）

| 信号 | 说明 |
|:----|:----|
| **创建时间 < 24 小时** | 极早期的代币风险最高 |
| **Holder 数量 < 100** | 没有真实用户基础 |
| **非公开销售（未上 DEX）** | 只能在开发者指定的网站购买 |
| **LP 未锁定** | 创建者随时可以撤走所有流动性 |
| **流动性池极低（< $10K）** | 几十万市值的假象，实则流动性只有几千 |
| **抹茶/薄饼之外的 DEX** | 未在主流通 DEX 上交易 |

### ⚠️ 中危信号（谨慎评估）

| 信号 | 说明 |
|:----|:----|
| **买卖税率不对等（买 1% 卖 10%+）** | 卖税过高可能是貔貅前兆 |
| **社交账号刚创建** | Twitter 账号创建不到 1 个月 |
| **Website 用模板建站** | 项目方投入极少 |
| **无审计报告** | 没有经过第三方审计 |
| **代币名称与知名项目极其相似** | 仿盘蹭热度（如 Pepa vs Pepe） |
| **社交媒体零互动** | Twitter 帖子只有机器人的点赞 |

### ✅ 相对安全的信号

| 信号 | 说明 |
|:----|:----|
| **合约已验证（Verified）** | 代码公开可查 |
| **所有权已放弃（Ownership Renounced）** | 创建者不再拥有控制权 |
| **LP Token 已锁定（Liquidity Locked）** | 至少锁定 6-12 个月 |
| **经过知名机构审计** | 如 CertiK、SlowMist、Hacken 等 |
| **持有者分散** | 最大持有者（除 LP 外）占比 < 5% |
| **无买卖税或税率对称** | 买卖税不超过 5% 且对称 |
| **交易量真实** | 有持续且分散的交易活动 |
| **主流 DEX 上交易** | Uniswap、PancakeSwap 等 |

## 五、发现貔貅币后怎么办？

### 5.1 被貔貅币套住了怎么办？

不幸已经买入了貔貅币，以下方法可以尝试（但成功率不高）：

1. **通过特定路由卖出**
   - 有的貔貅合约只阻止通过 DEX 路由卖出，但不阻止直接转账到第三方地址
   - 尝试将代币直接发送到另一个钱包地址，再从该地址尝试卖出

2. **调整滑点和 Gas**
   - 将滑点调高到 30-50%（如果卖出是「高税率」而非完全阻止，高滑点可能覆盖掉税率）
   - 尝试不同的 Gas 价格，绕过一些简单的检测

3. **使用 Uniswap V2 而非 V3**
   - 有些貔貅合约针对特定路由进行了限制
   - 尝试在合约支持的所有 DEX 上尝试卖出

4. **通过第三方聚合器**
   - 使用 1inch、ParaSwap 等聚合器，有时可以发现不同的路由路径

5. **等待开发者「临时开闸」**
   - 有些貔貅盘的开发者会在价格拉到足够高后，短暂开放卖出权限让「自己人」出货——但你能把握这个时机的概率几乎为零

> ⚠️ **残酷的现实**：如果以上方法都无效，你的资金大概率已经无法挽回。**不要投入更多资金试图「摊低成本」或「补仓」**——这只会让你亏得更多。**止损离场，及时总结教训。**

### 5.2 如何举报貔貅币

| 平台 | 举报方式 |
|:----|:--------|
| **DexScreener** | 在代币页面点击 Report，填写欺诈原因 |
| **区块浏览器** | 在 Etherscan/BscScan 上的代币页面点击 Report/Flag |
| **Token Sniffer** | 通过官网提交可疑合约地址 |
| **当地执法机构** | 如果损失巨大（>$10,000），建议通过当地网警报案 |

> 📌 在中文社区，你也可以到 [慢雾 SlowMist](https://slowmist.com) 的安全社区或 [链节点](https://www.chainnode.com) 的防骗板块发帖曝光，帮助更多人避免上当。

### 5.3 如何追溯貔貅盘创建者

虽然链上基本上是匿名的，但有时可以通过以下方式追溯：

1. **分析部署交易**：在区块浏览器的 Contract Creation TX 中，看创建者的地址（Deployer）
2. **关联地址分析**：在 [DeBank](https://debank.com) 或 [Etherscan](https://etherscan.io) 上搜索创建者地址，看它和其他哪些地址有交互
3. **资金流向追踪**：创建者是否将资金转入过 CEX 地址？如果找到了 CEX 地址，可以向 CEX 举报

**可用的链上分析工具：**
- [Etherscan](https://etherscan.io) — 基础查询
- [Debank](https://debank.com) — 多链地址追踪
- [Chainalysis](https://www.chainalysis.com/) — 专业工具，需经由执法机构使用
- [Arkham Intelligence](https://arkhamintelligence.com) — 可视化链上追踪

## 六、常见场景与案例分析

### 案例 1：经典 BSC 貔貅盘

**场景**：PancakeSwap 上出现了一个名为「SafeDogeMoon」的代币，图表显示开盘 2 小时涨了 50 倍。社区群里有几十人欢呼「上车」，「带单老师」在喊单。

**分析**：
- Token Sniffer 评分 22/100，标注为 Honeypot
- 合约未验证（No Source Code）
- 持有者分布：前 3 个地址持有 94% 的代币
- LP Token 在创建者地址中，未锁定

**结果**：买入的用户全部无法卖出。3 天后创建者撤走 $500K 流动性，代币归零。

**教训**：看似繁荣的社群聊天可能是「人设号」在演双簧——所有韭菜的钱都进了同一个池子。

### 案例 2：ETH 上的复杂貔貅

**场景**：一个新 Meme 币在 Uniswap V3 上线，Token Sniffer 评分 75/100，Honeypot.is 检测显示 No。看起来一切正常。

**分析**：
- 合约采用动态税率（buyTax = 1%, sellTax = 5%）——看起来正常
- 但合约包含 `addExcludedFromFees()` 函数——创建者可以免税卖出
- 合约中有 `setSwapAndLiquidityEnabled()`——创建者可以关闭流动性转换功能

**结果**：上线 2 天后，创建者调用 `setSellTax(100)` 将所有用户的卖出税率设为 100%，然后通过免税费地址卖出获利。

**教训**：即便通过了基础检测，如果合约中有「动态调整」功能（`onlyOwner` 可以修改的参数过多），风险依然很高。

## 七、预防胜于止损

### 7.1 建立投资纪律

```
✅ 加入 $1,000 以上的代币 → 必须通过 Token Sniffer + Honeypot.is 检测
✅ 小额投机（<$100）    → 也要做基础检测，但心态上接受「全亏」
✅ 新代币上线的前 24 小时  → 坚决不碰（让「聪明钱」先测试）
✅ 看到 1000% 涨幅时       → 先问自己「还轮得到我赚钱吗？」
✅ 设置一个每日/每周的「垃圾币预算上限」→ 亏完不再加钱
```

### 7.2 使用安全工具
