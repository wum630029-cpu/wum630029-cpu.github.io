## 一、Aave 是什么？

### 1.1 一句话理解

**Aave 是一个去中心化借贷协议——你可以把它理解为链上的「银行」。**

| 传统银行 | Aave |
|:--------|:----|
| 你把钱存银行赚利息 | 你把加密资产存入 Aave 赚利息 |
| 银行把你的钱借给别人 | Aave 把你的资产借给链上借款人 |
| 银行决定利率 | 利率由市场供需自动调节 |
| 有存款保险（部分国家） | 开源智能合约，无需信任中介 |

但 Aave 与传统银行有几个**本质区别**：

1. **完全去中心化**——没有 CEO、没有客服、没有 KYC，全部由智能合约自动执行
2. **透明公开**——每一笔交易、每一个利率都在链上可查
3. **任何人都可以参与**——只要你有钱包和加密资产
4. **创新的超額抵押机制**——借出资产需要提供超额抵押品

### 1.2 Aave 的核心功能

| 功能 | 说明 |
|:----|:----|
| **存款（Supply）** | 存入资产赚取浮动利息 |
| **借款（Borrow）** | 抵押资产后借出其他代币 |
| **利率切换** | 在浮动利率和稳定利率之间切换 |
| **闪电贷（Flash Loan）** | 无需抵押的无许可贷款（进阶功能） |
| **安全模块（Safety Module）** | 质押 AAVE 代币为协议提供安全 |

### 1.3 Aave 支持的链

Aave 目前已部署在多个主流公链上：

| 链 | Aave 版本 | 推荐用途 |
|:---|:---------:|:--------|
| **以太坊主网** | Aave V3 | 资金量大，生态最成熟 |
| **Base** | Aave V3 | Gas 低，适合新手体验 |
| **Arbitrum** | Aave V3 | L2，交易速度快 |
| **Optimism** | Aave V3 | L2，与 OP 生态结合 |
| **Polygon** | Aave V3 | 低费用，适合小额操作 |
| **Avalanche** | Aave V3 | 子网生态丰富 |

> 💡 **新手推荐**：从 **Base** 或 **Arbitrum** 开始——Gas 费低、操作体验好，用少量资金就能完整走一遍流程。

## 四、实操一：存入资产（Supply）赚取利息

我们先做最基础的操作——**存入资产到 Aave 获取利息**。

### 4.1 操作步骤

```
1️⃣ 在 Asset 列表中找到你要存入的资产（如 USDC）
2️⃣ 点击该行右侧的「Supply」按钮
3️⃣ 在弹出的窗口中，输入要存入的数量
4️⃣ 点击「Supply」→ 在 MetaMask 中确认授权（Approve）
   （第一次与该资产交互，需先授权 Aave 使用你的代币）
5️⃣ 再次点击「Supply」→ 在 MetaMask 中确认交易
6️⃣ ✅ 完成！你可以在 Dashboard 中看到你的存款
```

### 4.2 存入后会发生什么？

存入后，你会收到对应数量的 **aToken**（如 aUSDC）：

| 操作 | 你在 Aave 中的资产 |
|:----|:-----------------|
| 存入 1000 USDC | 钱包中 USDC 减少 1000，Aave 账户中显示 1000 USDC 存款 |
| 获得 aUSDC | aUSDC 代币进入你的钱包（代表你在 Aave 中的存款凭证） |

**aToken 有什么作用？**
- aToken 代表你在 Aave 中的存款份额
- **aToken 余额每天自动增长**——利息以 aToken 数量的形式直接发放
- 随时可以将 aToken 换回原始资产（点击 Withdraw）

### 4.3 取回资产

想取回资产时：

```
1️⃣ 在 Dashboard 的「Your Supplies」区域找到对应资产
2️⃣ 点击「Withdraw」
3️⃣ 输入要取回的数量
4️⃣ 确认 MetaMask 交易
5️⃣ ✅ 原始资产回到你的钱包
```

> 💡 **注意**：如果部分资产已被借出，你可能无法一次性取出全部——因为 Aave 需要确保池子中有足够的流动性。

### 4.4 存款利息怎么算？

Aave 的利率是 **动态变化的**：

| 状态 | 存款利率 | 借款利率 |
|:----|:-------:|:-------:|
| 池子资金充足（利用率低） | 较低 | 较低 |
| 池子资金紧张（利用率高） | 较高 | 较高 |
| 几乎被借光（利用率极高） | 很高 | 非常高 |

**利用率（Utilization Rate）= 总借款 / 总存款**

利用率越高，存款利率和借款利率都上升——这是一个自动调节的市场机制。

## 五、实操二：借出资产（Borrow）

存入资产后，你可以用它作为抵押来**借出其他代币**。

### 5.1 借出的意义

为什么要在 Aave 里借钱？——**这不是为了「借钱花」，而是为了释放流动性。**

| 场景 | 说明 |
|:----|:----|
| **持有 ETH 但又需要用 USDC** | 存入 ETH → 借出 USDC，不需要卖掉 ETH |
| **杠杆做多** | 存入 ETH → 借出 USDC → 去换更多 ETH → 再存入 Aave…… |
| **现货-合约套利** | 借出资产用于资金费率套利 |
| **紧急资金需求** | 临时需要 USDC 周转，用已有资产抵押借出 |

### 5.2 操作步骤

```
1️⃣ 在 Asset 列表中找到你要借出的资产（如 USDC）
2️⃣ 点击该行右侧的「Borrow」按钮
3️⃣ 选择利率类型（建议先选「Variable」浮动利率）
4️⃣ 输入要借出的数量
5️⃣ 系统会显示借出后的健康因子预估
6️⃣ 点击「Borrow」→ 在 MetaMask 中确认
7️⃣ ✅ 借出的资产立即进入你的钱包！
```

### 5.3 核心概念：你需要理解这三个数字

#### ① LTV（Loan-to-Value，贷款价值比）

**LTV = 借款价值 / 抵押品价值**

| 资产 | 最大 LTV |
|:----|:-------:|
| ETH | 80-82.5% |
| USDC | 80-82.5% |
| wstETH | 75% |
| WBTC | 73-75% |

**举例：**
- 存入价值 $10,000 的 ETH（最大 LTV = 80%）
- 你最多可以借出 **$8,000** 等值的其他代币

> 💡 **保守建议**：不要用到最大 LTV。将 LTV 控制在 30-50% 是比较安全的范围。

#### ② Liquidation Threshold（清算阈值）

**当 LTV 超过清算阈值时，你的抵押品会被清算。**

| 资产 | 清算阈值 | 清算惩罚 |
|:----|:-------:|:-------:|
| ETH | 85% | 5-10% |
| USDC | 85% | 5-10% |
| wstETH | 80% | 5-10% |

**继续上面的例子：**
- 存入 $10,000 ETH，借出 $8,000 USDC
- LTV = 80%，清算阈值 = 85%
- 如果 ETH 价格下跌导致 LTV 达到 85%，你的抵押品会被清算

#### ③ Health Factor（健康因子）

**Aave 界面最醒目的数字——它告诉你距离清算还有多远。**

```
Health Factor = 抵押品总价值 × 清算阈值 / 借款总价值

- HF > 2.0  → 非常安全
- HF 1.5-2.0 → 安全
- HF 1.1-1.5 → ⚠️ 需要密切关注
- HF < 1.0 → 🔴 你已经进入清算区域！
```

### 5.4 图示：LTV 与清算的关系

```
    安全区             警戒区            清算区
  ┌────────────┐   ┌──────────┐   ┌───────────┐
  │  LTV 0-70% │ → │ LTV 70%  │ → │ LTV ≥ 85% │
  │  HF > 1.2  │   │  至 85%  │   │ (清算)    │
  │  很安全    │   │ 注意风险  │   │ 会被强平   │
  └────────────┘   └──────────┘   └───────────┘
        ↑               ↑               ↑
   推荐新手范围     适度杠杆       不要进入的区域
```

## 七、风险管理：如何避免清算

**清算（Liquidation）是 DeFi 借贷中新手最容易踩的坑。**

当你的健康因子降到 1.0 以下时，任何人都可以触发清算——他们会偿还你的部分借款，并获取你的抵押品（含清算惩罚折扣）。

### 7.1 清算发生时你会损失多少？

| 清算比例 | 你的抵押品中会被清算的部分 |
|:--------|:-------------------------|
| Aave V3 (Base/Arbitrum等) | 最多 50% 的借款 |
| Aave V3 (以太坊主网) | 最多 50% 的借款 |

清算时，清算人会偿还你的一部分借款，并额外拿走最多 **5-10% 的清算惩罚**——这意味着你损失的不仅仅是价格波动的部分。

### 7.2 清算防御指南

```
🛡️ 清算预防清单

□ 保持 LTV < 50%
   → 即使被清算，也是部分清算（有缓冲）

□ 监控 Health Factor
   → 养成每天查看的习惯
   → HF < 2.0 时提高警惕
   → HF < 1.5 时主动补充抵押或还贷

□ 资产选择
   → 选择波动较小的资产作为抵押（USDC/BTC 比山寨币更稳定）
   → 不要用高波动资产做高 LTV 借款

□ 设置价格警报
   → 用 DeBank、Zapper 或 Telegram Bot 设置价格提醒
   → 在 CoinGecko/CoinMarketCap 设置通知

□ 预留 Gas 费
   → 钱包里永远留够用于补充抵押/还贷的 Gas 费
   → 行情剧烈波动时 Gas 费可能飙升
```

### 7.3 谁最容易触发清算？

> **2026 年的 DeFi 市场**中，大量清算者使用自动化机器人监控整个链——
> 一旦你的 HF 降到 1.0 以下，几乎在下一个区块就会被清算。
>
> 这不是「可能被清算」，而是**必然被清算**。没有侥幸。

## 八、利率切换与进阶操作

### 8.1 浮动利率 vs 稳定利率

Aave V3 提供两种借款利率：

| 利率类型 | 含义 | 适合场景 |
|:--------|:----|:--------|
| **浮动利率（Variable）** | 随市场供需变化而变化 | ✅ 大多数情况的默认选择 |
| **稳定利率（Stable）** | 相对稳定，但偶尔协议会调整 | 长期借款，锁定利率成本 |

**如何切换？**
- 在 Dashboard 的「Your Borrows」中找到已借出的资产
- 点击「APY」旁的小箭头 → 选择「Switch to Stable/Variable」
- 确认交易即可（这是一次链上操作，需付 Gas）

> 💡 **经验之谈**：对大多数用户来说，**浮动利率是更好的选择**——稳定利率在极端行情下可能被协议一次性大幅上调，失去「稳定」的意义。

### 8.2 闪电贷（Flash Loan）——概念了解

**闪电贷是 Aave 独创的杀手级功能：一笔无需任何抵押的贷款，前提是必须在同一笔交易中归还。**

```
闪电贷的核心机制：

┌─────────┐    借出 $100 万    ┌──────────┐
│  你的    │ ──────────────── → │  Aave    │
│  合约    │                   │  协议    │
│         │ ← ──────────────── │          │
└─────────┘  归还 $100 万+费   └──────────┘
               用（0.09%）

  关键：这两步必须在同一个以太坊交易中完成
  如果归还失败 → 整笔交易回滚 → 相当于什么都没发生
```

闪电贷的常见用途：

| 用途 | 说明 |
|:----|:----|
| **套利** | 利用不同 DEX 之间的价差瞬间套利 |
| **抵押品置换** | 在不还贷的情况下置换抵押品 |
| **债务重组** | 从一个借贷平台转到另一个 |
| **自我清算** | 有人要清算你时，自己先操作可以减少损失 |

> ⚠️ **闪电贷需要用 Solidity 智能合约来操作**——一般用户暂时不需要学习，但理解这个概念有助于了解 DeFi 的深度。

### 8.3 eMode（高效模式）

Aave V3 的新功能：**如果你存入和借出的是高度相关的资产（如 USDC → USDT，或 ETH → wstETH），可以使用 eMode 获得更高的 LTV。**

| 场景 | 普通模式 LTV | eMode LTV |
|:----|:----------:|:---------:|
| 存入 ETH 借出 wstETH | 80% | 高达 92% |
| 存入 USDC 借出 USDT | 80% | 高达 92% |

> 如果你只在稳定币之间操作或 LSD（流动性质押衍生品）之间操作，eMode 可以释放更多流动性。

## 九、Aave 与其他借贷协议的对比

| 协议 | 核心特点 | 适合人群 |
|:----|:--------|:--------|
| **Aave** | 最成熟，功能全面，支持 10+ 条链 | ✅ 所有用户 |
